## 2. IP-Adapter 混合模式實作

### 架構決策
原本角色設計生成只有「VLM 看圖→萃取文字→SD prompt」這一條路。  
評估後採用**混合模式**：VLM 文字萃取保留，同時上傳概念圖作為 IP-Adapter visual anchor。

| 路線 | 作用 |
|---|---|
| VLM（Qwen）| 萃取外觀描述 → 補進中文 raw_desc → 翻譯成 SD tag |
| IP-Adapter | 直接注入圖像 embedding → UNet cross-attention → 外觀視覺一致性 |

### 後端改動（art_generate.py）

**`_inject_loras` 修正**：原本只 rewire `KSampler.model`，但 `ipadapter_txt2img.json` 的 KSampler 吃 `IPAdapterAdvanced` 輸出，不直接吃 checkpoint。加入：
```python
elif ct == "IPAdapterAdvanced":
    if isinstance(inputs.get("model"), list) and inputs["model"][0] == ckpt_id:
        inputs["model"] = [prev_id, 0]
```
LoRA chain 正確接入：`last_lora → IPAdapterAdvanced → KSampler`。

**`generate_character_design` / `generate_variant_design`**：
- 新增 `use_ipa: bool = True` query param
- `use_ipa=True` 且有可讀概念圖 → 上傳第一張到 ComfyUI → 切換 `ipadapter_txt2img.json`
- `use_ipa=False` 或無概念圖 → 維持 `text_to_image.json`
- Response header 加 `X-IPA-Used: 1/0`

### 前端改動（CharacterTab.jsx）
- `ipaEnabled` state（主角色）/ `_initVariant` 加 `ipaEnabled: true`（變體）
- 生成按鈕旁加「概念圖參考」checkbox（藍色），預設勾選
- `generateDesignImage` / `generateVariantDesignImage` 帶 `use_ipa` query param
- Debug panel 加 `IP-Adapter ON/OFF` badge

### IP-Adapter 啟用邏輯（最終版）
```
use_ipa=ON（checkbox 勾選）AND 有上傳概念圖 AND 圖片可讀取
    → 啟用，使用第一張（草圖也接受）
use_ipa=OFF 或無概念圖
    → 純文字生成
```
原本的 `_is_flat_color_draft` 偵測仍保留用於 VLM 分析的 flat/color 標記，**不再影響 IP-Adapter 啟用**。

## 3. AI 提示詞 toggle 行為修正 + 服裝設定欄位

### 問題（AI 提示詞）
取消勾選後按儲存 → 寫入 `null` → 下次進入欄位內容消失。

### 修正方向
- **儲存**：永遠寫入文字（`aiPrompt.trim() || null`），不受 checkbox 影響
- **生成是否使用**：由 `use_ai_prompt` query param 傳給後端，由 checkbox 當下狀態決定
- Checkbox 成為「生成時使用 / 不使用 + 顯示 / 隱藏」的雙重開關

### 服裝設定（新欄位）

**後端**：
- `character.py` 加 `outfit: Mapped[Optional[str]]`
- `schemas/character.py`：CharacterCreate / Update / Response 加 `outfit`
- `database.py`：migration 加 `("outfit", "TEXT")`
- `characters.py`：`_EMPTY_VARIANT` 加 `"outfit": None`；`VariantUpdate` 加 `outfit`
- `art_generate.py`：`generate_character_design` / `generate_variant_design` 的 `raw_desc` 加入 `服裝設定：{outfit}`（受 `use_outfit` query param 控制）

**前端**：
- 主角色：`outfit` / `outfitEnabled` state；服裝 UI（checkbox + textarea）；save 永遠帶文字
- 變體：`_initVariant` 加 `outfit` / `outfitEnabled`；variant UI 同上；`saveVariantFields` 帶 outfit
- 生成時傳 `use_outfit=1/0` query param

## 4. 語義特徵注入 + 幻覺阻斷 Negative 注入

### 使用者改動（已 commit）

**`lexicon.py`** — 新增 `TRAIT_MAP` / `TRAIT_RE`：
```python
TRAIT_MAP = {
    "下垂眼": "drooping eyes, tareme",
    "垂眼": "drooping eyes, tareme",
    "貓眼": "cat eyes",
    "大小姐": "noble female, ojou-sama",
    "窄裙": "tight skirt",
    "溫柔": "gentle expression",
    "黑絲": "black thighhighs",
    "黑絲襪": "black thighhighs",
}
```

**`compiler.py`** — `_inject_traits()`：掃描原文，找到 TRAIT_MAP 關鍵詞就強制插入對應 tag；大小姐偵測到 → 從 positive 刪除女僕相關 tag。

**`art_generate.py`** — `_age_body_tags()` 調整：≤12 改用 `child, young girl,...`。

⚠️ **待確認**：`"young girl"` / `"teenage girl"` 等詞在男性角色下會造成性別衝突（與 `1boy` 矛盾），建議之後加 gender 參數傳入，或改為性別中性詞。

### 補強：Negative 側注入（Claude 加入）

原本 `_inject_traits` 只從 positive 刪除幻覺 tag，diffusion 過程中概念仍可能殘留。加入：

**`_CONTEXT_BLOCKERS`** 規則表（可擴充）：
```python
_CONTEXT_BLOCKERS = [
    (
        {"noble female", "ojou-sama", "princess"},   # 觸發條件
        {"maid", "maid outfit", ..., "apron"},        # 從 positive 移除
        "maid, maid outfit, maid headdress, apron, waitress",  # 壓進 negative
    ),
]
```

**`_inject_traits` 回傳值改為 `tuple[list[str], str]`**（positive_tags, extra_negative_str）。

**`compile()`** 把 extra_negative 合入最終 negative prompt：
```python
if _extra_neg:
    negative = f"{negative}, {_extra_neg}" if negative else _extra_neg
```

效果：CFG guidance 主動推離女僕方向，不只刪 positive tag。
