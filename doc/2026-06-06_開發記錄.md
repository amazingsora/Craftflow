## 今日修改

### 1. Logging 修正（`main.py`）

`main.py` 缺少 `logging.basicConfig`，導致應用層 log 全部丟棄。
新增後可在後端 CMD 視窗看到 `[combined-vision]`、`[char-gen]` 等詳細 log。

### 2. Coverage 偵測 prompt 修正（`art_generate.py`）

舊定義「腿可見 = full」導致半身草圖被誤判為 full。

| 等級 | 舊定義 | 新定義 |
|------|--------|--------|
| full | 腿和腳都可見 | **腳踝和腳都清晰可見（完整全身）** |
| partial | 身軀可見但腿被切掉 | **膝蓋或腳踝以下被切掉（大腿可見但無腳也算 partial）** |
| bust | 只有臉/肩膀 | 只有腰部以上可見 |

同樣修正 `_detect_body_coverage`（fallback 函式）。

### 3. 嘗試歷程 A–E（均失敗或有問題）

見 `doc/改善構圖需求 .md` 詳細記錄。

### 4. 當前方案：partial/bust → CN bypass + IPA only

```python
# art_generate.py（char-gen & variant-gen）
if coverage == "full":
    _cn_ref_bytes = _shrink_for_full_body(...)   # 維持原邏輯
else:
    _cn_ref_bytes = None                          # CN bypass
    # IPA 仍作用，外觀由 IPA + text prompt 引導
```

結果：
- ✅ 單一全身角色，無設計稿格式
- ✅ 無接縫，風格統一
- ⚠️ 姿勢不跟草圖，IPA weight 低時外觀相似度偏低

### 5. 新增的 helper 函式（備用，暫不呼叫）

- `_create_inpaint_canvas_and_mask()` — 草圖放上半 + 生成 inpaint mask
- `_inject_inpaint_nodes()` — 注入 VAEEncodeForInpaint（惡縫方案）
