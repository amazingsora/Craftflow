# 2026-05-26 開發記錄

## 今日工作：繪圖側 UX 優化（5 項）

### 背景

以消費者（創作家）角度審視現有功能後，識別出繪圖側的工作流斷裂與體驗問題，優先於文字/小說側進行修整。

---

## 1. 草圖問答 → 送到生圖

**問題：** ComposeTab 產出的 `suggested_prompt` 需要手動複製貼到 GenerateTab，流程斷裂。

**做法：**
- `App.jsx` 新增 `pendingGenPrompt` state 與 `onSendToGenerate(promptZh)` handler；觸發時自動切換至 generate tab。
- `ComposeTab` 接收 `onSendToGenerate` prop，在 SDXL Prompt 標題旁加「→ 送到生圖」按鈕；帶入的內容是使用者的**問題描述**（`question`），而非 SDXL 英文 prompt，讓使用者可在生圖 tab 進一步編輯再 AI 編譯。
- `GenerateTab` 接收 `pendingPrompt` + `onPromptConsumed` props；透過 `useEffect` 在收到 pendingPrompt 時寫入 `promptZh` 並清空 `optimizedEn`，然後呼叫 onPromptConsumed 清除 App 端 state。

---

## 2. 角色卡 → 以此角色生圖

**問題：** 角色管理和生圖工具完全孤立，角色設定無法直接驅動生圖。

**做法：**
- `CharacterTab` export 接收 `onSendToGenerate` prop，向下傳至 `CharacterDetailView`。
- `CharacterDetailView` 在 toolbar 右側加「→ 以此角色生圖」按鈕，點擊時組合 `name + core_traits + outfit` 為中文描述字串送至 GenerateTab。
- 生成字串格式：`{name}，{core_traits}，服裝：{outfit}`（有欄位才加）。

---

## 3. 進度條改為 indeterminate 動畫

**問題：** ComposeTab 和 GenerateTab 的進度條是基於固定 timer 的假進度，會在 ComfyUI 卡住時顯示 95% 靜止不動，傳達錯誤資訊。

**做法：**
- 移除兩個 Tab 的 `progress` state 和 `progressTimer` ref。
- 保留 `elapsed` 計時器（真實耗時）。
- 進度條改為 CSS `indeterminate` 動畫（左→右掃描，無百分比），在 `index.css` 加入 `@keyframes indeterminate`。
- ComposeTab 顯示文字：「AI 分析草圖 + ComfyUI 生成參考圖...」+ 已耗時。
- GenerateTab 顯示文字：「ComfyUI 正在繪圖中...」+ 已耗時。

---

## 4. 歷史記錄加類型過濾器

**問題：** 100 張縮圖橫向排列，無法快速找到特定類型的圖。

**做法：**
- `App.jsx` 新增 `historyFilter` state（預設 `'all'`）。
- 在歷史記錄標題列下方，有圖時顯示過濾按鈕列：**全部 / 線稿 / 生圖 / 問答 / 角色**。
- `'generate'` filter 涵蓋 `['generate', 'i2i', 'controlnet', 'ipadapter']` 四種類型。
- 把 `history.map(...)` 改為對 `filtered` 陣列 render（IIFE 模式，避免在 JSX 層宣告變數）。

---

## 5. LoRA 訓練移出主 Tab 列 → 設定 Modal

**問題：** LoRA 訓練是一次性設定操作，混在每日使用的 Tab 列中造成視覺雜訊。

**做法：**
- 從 `TABS` 陣列移除 `{ id: 'training', label: 'LoRA 訓練' }`，主 Tab 由 6 個縮減至 5 個。
- 在設定 Modal 底部加分隔線 + `TrainingTab` 元件。
- Modal `maxWidth` 從 520 擴至 560，容納訓練 UI。
