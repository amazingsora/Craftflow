## 背景：Gemini 的 `expand_full_body` 實作（作廢）

Gemini 在 branch `602gemini` 新增「補全下半身」checkbox，邏輯為：

- 前端傳 `expand_full_body=1` 給後端
- 後端以 `_pad_for_full_body()` 把 CN 參考圖底部填白，撐到全身畫布比例
- 期望：SD 能「延伸」上半身到全身

**根本無效的兩個原因：**

1. **概念錯誤**：Canny ControlNet 只約束邊緣結構，底部白色代表「無邊緣」，SD 自由填充，但不知道要「延伸」現有角色，與原圖無延續性。
2. **函式互相抵銷**：`_pad_for_full_body` 把圖 pad 到 `_fullbody_canvas()` 的尺寸；後續 `_letterbox_to_aspect` 發現比例完全相符 → 直接 `return image_bytes` 原樣回傳，等於 no-op。

---

## 真正的問題

使用者上傳的概念草圖（上半身到大腿）**佔滿了畫框**。Canny ControlNet 照著這個構圖約束 SD，導致生成圖同樣被裁切、看不到腳。

解法方向：**縮小 CN 參考圖**在畫框內的比例，讓 SD 有空間在下方生成腳部，而不是「補全圖片」。

## 本日改動：視覺自動偵測 + CN 參考縮放

### 新增函式（`backend/app/api/art_generate.py`）

**`_detect_body_coverage(image_bytes) -> str`**
- 呼叫 `_oc.analyze_image_bytes(_, model=state.get_vision_model())`（不硬編碼模型）
- prompt 要求視覺模型分類：`"full"` / `"partial"` / `"bust"`
- 含 fuzzy match 容錯，失敗 fallback 為 `"partial"`

**`_shrink_for_full_body(image_bytes, target_w, target_h, coverage) -> bytes`**
- `"full"` → 原樣回傳（交由既有 `_letterbox_to_aspect` 處理）
- `"partial"` → 縮到畫布高 72%，偏上 4% 置中貼入
- `"bust"` → 縮到畫布高 55%，偏上 5% 置中貼入
- 背景色沿用 `_border_color()`（與 letterbox 一致，避免接縫）

### 修改端點

`generate_character_design` 和 `generate_variant_design`：

- 移除 `expand_full_body: bool = False` 參數
- 全身模式（`not is_expression`）且 ControlNet 啟用時，自動呼叫偵測 → 縮放
- 已替換 `_pad_for_full_body`，無殘留引用

### 前端（`CharacterTab.jsx`）

- 移除 `expandFullBody` state、「補全下半身」checkbox、`expand_full_body` API 參數
- `_initVariant` 中對應 prefs key 一併清除

## 待確認問題

### CN 權重 ~0.9 無法生成全身圖

使用者觀察：CN weight 設在 0.9 附近時，生成結果會被裁切（無腳），降到較低值才能完整。

**推測根因（尚未驗證）：**
- CN 強度過高時，SD 嚴格跟隨 Canny 邊緣 → 參考圖是上半身構圖 → 強制複製構圖比例
- 縮小後的 CN 參考圖（本次改動）應有改善，但可能仍不足
- 可能需要 `cn_weight` 上限調整，或在全身模式自動降低 CN 權重（如 full body mode → weight × 0.85）

**明日驗證：** 以新的 `_shrink_for_full_body` 邏輯生成一次，觀察不同 CN weight 下的效果。
