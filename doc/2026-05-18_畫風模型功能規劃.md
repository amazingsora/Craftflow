## 功能目標

讓使用者可以建立、管理、套用自訂「畫風模型」。

### 雙路線並行

| | 路線 A — 組態式畫風 | 路線 B — 個人風格訓練 |
|---|---|---|
| **原理** | checkpoint + 社群 LoRA + prompt 設定打包 | 用自己的作品訓練個人 LoRA |
| **需要作品** | 不需要 | 5 張起步，目標 30 張 |
| **效果上限** | 取決於社群 LoRA 品質 | 可精準模仿個人筆觸 |
| **何時可做** | 現在（Phase 1–6） | 邊開發邊累積作品（Phase 7） |

路線 B 採**漸進測試**策略：5 → 10 → 20 → 30 張，每個量級都訓練一次，觀察品質提升曲線。

### 畫風模型的內容

- Base Style（決定 LLM 編譯模板：sdxl / pony / flux / noobai / illustrious / anythingxl）
- Quality prefix / Negative prompt（空白 = 繼承 Base Style 預設）
- Extra Tags（固定附加在每次 positive prompt 尾端）
- LoRA 模型（社群 LoRA 或個人訓練 LoRA，可多個，各自設 weight）

建立後可套用到**角色**或**專案**，生成時自動使用，不再依賴全域 checkpoint 切換。

## 現有架構對應

| 現有元件 | 說明 |
|---|---|
| `checkpoint_styles.yml` | checkpoint 名稱 → PromptStyle 映射，保留不動 |
| `PromptStyle` enum | 固定 6 種 style，保留不動 |
| `STYLE_CONFIG` | 每種 style 的預設值，`ArtStyle` 是其上的覆寫層 |
| `state.py` `get/set_checkpoint()` | 全域執行期 checkpoint，保留作為 fallback |
| `art_generate.py` `_load_workflow()` | 注入 checkpoint，Phase 3 在此基礎上再注入 LoRA |

### ✅ Phase 2 — Prompt Engine 整合

**覆寫規則（已實作）：**

```
effective quality_prefix = art_style.quality_prefix (若非空) else STYLE_CONFIG[base_style]["quality_prefix"]
effective negative       = art_style.negative       (若非空) else STYLE_CONFIG[base_style]["negative"]
effective base_style     = art_style.base_style     (若有)   else _detect_style() (讀 checkpoint)
extra_tags               = 附加在 final_positive 尾端
```

**異動檔案：**

| 檔案 | 說明 |

**受影響端點（加入 `art_style_id` 支援）：**

| 端點 | 接收方式 |
|---|---|
| `POST /art/compile-prompt` | Request body `art_style_id: int \| null` |

### ✅ Phase 3 — LoRA 注入（ComfyUI Workflow）

**異動檔案：**

| 檔案 | 說明 |
|---|---|
| `backend/app/api/art_generate.py` | 新增 `_inject_loras(wf, loras)` helper；在 3 個生成端點的 `_load_workflow()` 之後呼叫 |
| `backend/app/api/settings.py` | 新增 `GET /settings/loras` 端點 |
| `frontend/src/components/ArtStyleTab.jsx` | LoRA model 欄位加 `<datalist>` autocomplete，從 `/settings/loras` 動態載入 |

**`_inject_loras()` 邏輯：**

1. 過濾空 model 名稱（靜默跳過）
2. 找到 `CheckpointLoaderSimple` 節點 ID 作為鏈起點
3. 逐一插入 `LoraLoader` 節點（`_lora_0`, `_lora_1`, ...），每個的 model/clip 接上一個節點輸出
4. 重新連接所有 `CLIPTextEncode.clip` 和 `KSampler.model` 到最後一個 LoRA 節點
5. 若 workflow 無 `CheckpointLoaderSimple`（如 lineart），log warning 並跳過

**受影響端點（加入 LoRA 注入）：**

| 端點 | 說明 |
|---|---|
| `POST /art/generate` | 注入 art_style.loras |
| `POST /characters/{id}/generate-design` | 注入 art_style.loras |
| `POST /characters/{id}/variants/{slot}/generate-design` | 注入 art_style.loras |

`POST /art/compose` 不支援 art_style，不注入。`POST /art/lineart` 無 CheckpointLoaderSimple，自動跳過。

**前端行為：**
- ComfyUI 有 LoRA 時：輸入框顯示 autocomplete 下拉（`<datalist>`），仍可手動輸入任意檔名
- ComfyUI 未連線時：靜默失敗，退化為純手動文字輸入

**完成標準：**

- [x] `_inject_loras()` 函數可正確插入單一 LoRA 節點
- [x] 多個 LoRA 串接順序正確
- [x] LoRA 名稱為空時靜默跳過（不 crash）
- [x] `GET /api/v1/settings/loras` 回傳可用 LoRA 清單
- [x] 前端 LoRA 欄位支援 autocomplete，ComfyUI 未連線時退化為手動輸入

### 🔲 Phase 7 — 個人風格 LoRA 訓練（路線 B）

**目標：** 在 Craftflow 內管理個人 LoRA 訓練任務。

#### 訓練工具

**kohya_ss**（宿主機 Windows 安裝）— 與現有 SDXL / Illustrious checkpoint 相容。
> 訓練需要直接存取 GPU，不放在 Docker 內。

#### 訓練流程

```
1. 作品準備（Craftflow UI）
   - 上傳訓練圖片（PNG，1024×1024 以上）
   - Ollama vision 自動生成 caption 草稿
   - 人工確認 / 修正 caption

2. 訓練設定（Craftflow UI）
   - 選擇基底 checkpoint、trigger word
   - 設定 LoRA rank（預設 32）、學習率、epochs
   - 綁定輸出到哪個「畫風模型」

3. 訓練執行
   - 後端寫出 kohya_ss config.toml，subprocess 啟動
   - SSE 即時回傳 loss / step 進度

4. 完成
   - LoRA .safetensors 自動複製到 ComfyUI models/loras/
   - 自動掛載到綁定的畫風模型
   - 用 Phase 6 測試生成驗證效果
```

> 架構決策、技術細節、待確認事項詳見：`doc/2026-05-19_開發記錄.md` § Phase 7 技術調查

#### 漸進測試計畫

| 量級 | 目的 |
|---|---|
| 5 張 | 驗證 pipeline 能跑通，不期待品質 |
| 10 張 | 確認風格特徵是否被捕捉 |
| 20 張 | 主要品質評估點 |
| 30 張 | 穩定輸出目標 |

每個量級都訓練一次，用相同 prompt 對比輸出，記錄品質曲線。

#### 完成標準

- [ ] 可設定訓練參數並啟動 kohya_ss
- [ ] 訓練進度 SSE 即時顯示

## 技術注意事項

### 不破壞現有流程

- `PromptStyle` enum 和 `STYLE_CONFIG` 不動
- `checkpoint_styles.yml` 映射邏輯保留
- 所有 `art_style_id` 參數皆為 optional，不傳就走現有邏輯

### Phase 3 LoRA 注入（已解決）

- ComfyUI `/object_info/LoraLoader` 確實有 LoRA 清單（與 checkpoint 相同結構）
- `lineart` workflow 無 `CheckpointLoaderSimple`，`_inject_loras()` 自動跳過，不 crash
- LoRA 節點 ID 使用 `_lora_0`, `_lora_1`, ... 不會與原有整數 ID 衝突

### Phase 7 kohya_ss 安裝

- 安裝在 **宿主機 Windows**，不在 Docker
- 訓練輸出目錄需手動或自動同步到 ComfyUI 的 `models/loras/`
- subprocess 呼叫時注意路徑引號（Windows 路徑含空格）
