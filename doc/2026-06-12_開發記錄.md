### 重要發現：CRLF/LF 換行符

- repo 內為 LF、Windows 工作目錄為 CRLF（clone 時 autocrlf 轉換），沙箱 git 無 autocrlf 設定
  → 沙箱側 `git status` 會顯示**全部檔案 modified（+52,280/−52,280）**，為假象
- 沙箱側 commit 須先把要提交的檔 `sed 's/\r$//'` 正規化再 add，否則會把整檔改成 CRLF 入版
