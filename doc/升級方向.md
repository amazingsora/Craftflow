使用枚舉（Enum）優化特定字段：



許多字段（如 gender, status, report\_type, training\_job.status）目前是 String 类型。使用 Python 和資料庫層面的枚舉可以提高代碼的安全性和可維護性。例如：



Python

from enum import Enum as PyEnum

from sqlalchemy import Enum



class Gender(PyEnum):

&#x20;   MALE = "male"

&#x20;   FEMALE = "female"

&#x20;   NEUTRAL = "neutral"

&#x20;   OTHER = "other"



\# 在 Character 模型中

gender: Mapped\[Gender] = mapped\_column(Enum(Gender), nullable=True)

這能防止手誤（例如將 "female" 寫成 "Femal"），並使查詢更可靠。



默認值的一致性：



代碼中 JSON 字段的默認值有時是 list（Callable），有時是 None（tab\_names, variants）。這在 SQLAlchemy 中可能會導致行為不一致。建議統一使用 Callable，例如 default=list 或 default=dict。對於 tab\_names 和 variants 這種可選字段，應確保設置為 default=list 或提供一個清晰的處理方式。
