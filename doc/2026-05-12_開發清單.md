🚀 Craftflow 技術開發進度報告 (2026-05-12)
📋 任務概述
今日核心目標為解決 Craftflow Backend (Docker) 與 ComfyUI (Windows Host) 之間的整合問題，確保線稿生成功能（Lineart Generation）能全線跑通。

🛠️ 已解決的技術坑點
1. 檔案路徑混亂 (FileNotFoundError)
現象：後端無法讀取 sketch_to_lineart.json 工作流檔案。

原因：Docker 容器內的目錄結構層級較深，程式碼原先預測的路徑跳過了 tools/Craftflow/ 層級。

修正：將 art_generate.py 中的工作流目錄更新為容器內的絕對路徑：
_WORKFLOW_DIR = Path("/app/tools/Craftflow/diffusion/workflows")

2. Docker 跨環境通訊 (Connection Error)
現象：後端噴出 Network is unreachable 或 Connection refused。

原因：

容器內的 localhost 指向容器自身，而非 Windows 主機。

ComfyUI 預設僅監聽本地 127.0.0.1，拒絕外部連線。

修正方案：

後端設定：將 API 連線位址改為 http://host.docker.internal:8188。

Docker Compose：確保 backend 服務包含 extra_hosts 設定以解析主機網域。

ComfyUI 啟動：在啟動指令後方加上 --listen 0.0.0.0，開啟「公海模式」。

3. 硬體效能確認
驗證結果：成功偵測到主機端 NVIDIA GeForce RTX 5070 Ti 顯示卡，VRAM 約 16GB。

狀態：連線測試（/system_stats）回傳 JSON 數據正常，代表後端具備調度 GPU 算力的能力。
