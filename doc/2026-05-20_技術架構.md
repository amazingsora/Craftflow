# Craftflow — 技術架構文件

**版本：v1.0**
**建立日期：2026-05-20**

---

## 1. 系統概覽

Craftflow 是一套**本地優先**的個人創作輔助平台，結合小說管理與插畫生成。
AI 運算完全在本機執行，不依賴雲端服務。

```
瀏覽器 (localhost:3000)
    ↕  HTTP / SSE
React Frontend (Vite dev server)
    ↕  HTTP proxy → /api/*
FastAPI Backend (localhost:8000)
    ├── Ollama (localhost:11434)   ← 文字生成 / 視覺分析
    ├── ComfyUI (localhost:8188)   ← 圖片生成 / 線稿處理
    └── kohya_ss (subprocess)     ← LoRA 訓練
```

| | PyTorch 2.13+cu132 | GPU 加速（RTX 5070 Ti） |

---

## 3. 架構原則

1. **Local-First** — Ollama / ComfyUI 皆用 `localhost`（本機執行）
2. **Human-in-the-Loop** — AI 不覆寫原始創作內容，產出進分析報告
3. **Resilient** — Ollama / ComfyUI 失敗不 crash 後端，回傳明確錯誤
4. **Progressive feedback** — 長時間 AI 任務以 SSE 即時推送進度

## 8. 提示詞引擎（Prompt Engine）

位置：`backend/app/services/ai/prompt_engine/`

### 流程

```
中文輸入
    ↓
[1] LLM 翻譯（Ollama）— 依 style 選模板（styles.py）
    ↓
[2] 萃取 Output（_extract_output）
    ↓
[3] Sanitizer — 移除 banned_tags（自動含 quality_prefix tags）
    ↓
[4] Anchor 提取（_extract_color_anchors）
    ← 從中文原文 regex 萃取「髮色、眼色、髮長」
    ← anchor_text 優先（core_traits），避免 vision 污染
    ↓
[5] 衝突清除（_remove_conflicting_tags）
    ← 移除 LLM 輸出中與 anchor 衝突的髮色/眼色/髮長標籤
    ↓
[6] 排序 + 加權（_reorder_tags）
    ← Subject > (anchor:1.1) > Others > Meta
    ↓
[7] 拼接 quality_prefix（可被 ArtStyle 覆寫）
    ↓
(positive_prompt, negative_prompt)
```

### 支援的 Style（PromptStyle enum）

| Style | quality_prefix | 用途 |
|---|---|---|
| `sdxl` | masterpiece, best quality, high quality | 通用 SDXL |
| `pony` | score_9, score_8_up, score_7_up | Pony Diffusion |
| `flux` | （無）| Flux（自然語言格式） |
| `noobai` | masterpiece, best quality, newest, absurdres | NoobAI |
| `illustrious` | masterpiece, best quality, newest, highres | Illustrious XL |
| `anythingxl` | newest, masterpiece, best quality | Anything XL |

### ArtStyle 覆寫層

```
effective base_style     = art_style.base_style     (若有) else _detect_style()
effective quality_prefix = art_style.quality_prefix (若非空) else STYLE_CONFIG 預設
effective negative       = art_style.negative       (若非空) else STYLE_CONFIG 預設
extra_tags               = 附加在 positive 尾端
loras                    = 注入到 ComfyUI workflow（_inject_loras）
```

## 9. LoRA 訓練 Pipeline

### 架構

```
Frontend TrainingTab
    → POST /training/jobs/{id}/start
        ↓
Backend (training.py)
    1. 寫 caption .txt（與圖片同名）
    2. config_writer.write_config() → train_config.toml
    3. asyncio.create_task(_run_training)
        ↓
TrainingRunner（get_runner() factory）
    ├── LocalSubprocessRunner（TRAINING_RUNNER_MODE=local）
    │   subprocess → python C:\kohya_ss\sd-scripts\train_network.py
    │   解析 stdout 進度行（正則：steps: X%... Y/Z avr_loss=...）
    │   每 10 step 寫入 DB（current_step, last_loss, log_tail）
    │   SSE 廣播 → 前端進度條
    │
    └── RemoteAgentRunner（TRAINING_RUNNER_MODE=remote，Docker 備用）
        HTTP POST host.docker.internal:7788/train
        SSE GET  host.docker.internal:7788/progress/{id}

訓練完成
    → 複製 .safetensors → COMFYUI_LORAS_DIR
    → 若有綁定 ArtStyle → 自動加入 loras JSON
    → job.status = "done"
```

### 訓練圖片結構

```
data/training_images/
    img001.png
    img001.txt   ← "trigger_word, 1girl, silver hair, ..."（caption）
    img002.png
    img002.txt
    ...

lora_output/job_{id}/
    train_config.toml
    trigger_word-000010.safetensors   ← 每 N epoch 存一個
    trigger_word.safetensors          ← 最終輸出
```

## 13. 六大功能詳解

### 功能 1：草稿 → 線稿（ProcessTab）

**功能說明**
上傳手繪草圖，透過 ComfyUI 的 ControlNet 流程轉換為乾淨的線稿。適合在上色前整理線條或作為生圖參考。

**應用技術**
| 技術 | 用途 |
|---|---|
| ComfyUI | 執行 ControlNet lineart 工作流 |
| ControlNet | 保留草圖結構，萃取線條 |
| Lineart Preprocessor | 圖像前處理，強化邊緣 |

**操作流程**
1. 拖曳或點擊上傳草稿圖（JPG / PNG / WebP）
2. 確認左側預覽正確
3. 點擊「生成線稿」
4. 等待 ComfyUI 處理（約 30–90 秒）
5. 右側顯示線稿結果
6. 點擊「下載圖片」儲存

**相關 API**
- `POST /api/v1/art/lineart`（multipart，傳送圖片檔案）

---

### 功能 2：文字 → 生圖（GenerateTab）

**功能說明**
輸入中文描述，AI 自動翻譯並編譯為對應模型的 Danbooru 提示詞，再由 ComfyUI 生成圖片。支援種子固定、尺寸調整、Negative Prompt 自訂。

**應用技術**
| 技術 | 用途 |
|---|---|
| Ollama `dolphin-llama3` | 中文 → Danbooru tag 翻譯 |
| Prompt Engine（compiler.py） | Anchor 提取、衝突清除、排序加權 |
| ComfyUI | txt2img 生圖 |
| StyleConfig（styles.py） | 依 checkpoint 類型選擇 LLM 模板與 quality_prefix |

**操作流程**
1. 輸入中文描述（如「銀髮紫瞳的魔法師少女」）
2. 點擊「✨ AI 編譯提示詞」→ 自動翻譯並偵測畫風
3. 可手動補充英文技術 tag
4. 預覽最終 Prompt 組合（正面 + 負面）
5. 調整尺寸、Steps（10–50）、Seed（-1 = 隨機）
6. 點擊「生成圖片」
7. 進度條即時更新，完成後顯示結果
8. 下載或更換 Seed 重生

**相關 API**
- `POST /api/v1/art/compile-prompt`（取得編譯後 prompt）
- `POST /api/v1/art/generate`（生圖，支援 `art_style_id`）

---

### 功能 3：草圖問答（ComposeTab）

**功能說明**
上傳草圖並提出構圖問題（視角、光源、佈局等），Ollama 視覺模型分析後給出文字建議，並同步生成一張構圖參考圖。

**應用技術**
| 技術 | 用途 |
|---|---|
| Ollama `qwen2.5vl:7b` | 視覺理解 + 構圖建議生成 |
| Ollama `dolphin-llama3` | 將建議轉換為 Prompt |
| ComfyUI | 生成構圖參考圖 |

**操作流程**
1. 上傳草圖
2. 輸入構圖問題（範例：「這個視角的光源應該放哪裡？」）
3. 點擊「生成意見 + 參考圖」
4. 左欄顯示 AI 文字構圖建議
5. 右欄顯示 ComfyUI 生成的參考圖及對應 Prompt
6. 下載參考圖

**相關 API**
- `POST /api/v1/art/compose`（multipart，圖片 + 問題文字）

---

### 功能 4：角色管理（CharacterTab）

**功能說明**
完整的角色資料庫，支援多專案、多角色、最多 3 個服裝/表情變體。角色可綁定預設畫風，生圖時自動套用。勢力系統管理角色間的組織關係。

**應用技術**
| 技術 | 用途 |
|---|---|
| SQLite（characters / factions 表） | 角色資料持久化 |
| Ollama `dolphin-llama3` | AI 角色摘要生成 |
| Ollama `qwen2.5vl:7b` | 概念圖視覺分析 |
| Prompt Engine | core_traits 作為 anchor_text 強化髮色/眼色準確度 |
| ComfyUI | 人設圖 / 變體圖生成 |
| ArtStyle 優先鏈 | 手動指定 > 角色預設 > 專案預設 > checkpoint 自動偵測 |

**操作流程**
1. 建立或選擇專案
2. 新增角色（名稱、性別、身份、描述、外貌、個性）
3. 上傳概念圖（手繪參考）或點「生成人設圖」
4. 點「✨ AI 概括」自動生成角色摘要（寫入 ai_summary 欄位）
5. 設定「預設畫風」（生圖時自動套用，不需每次手選）
6. 新增最多 3 個變體（不同服裝/場景），各自可有獨立生成設定
7. 進入「勢力」分頁，建立勢力並將角色加入成員

**相關 API（主要）**

### 功能 5：畫風管理（ArtStyleTab）

**功能說明**
管理自訂畫風預設，每個畫風打包：基底模型類型、Quality Prefix、Negative Prompt、固定附加 tag、多個 LoRA 組合。建立後可套用到角色或專案，生圖時自動注入設定。

**應用技術**
| 技術 | 用途 |
|---|---|
| SQLite（art_styles 表） | 畫風設定持久化，loras 欄位以 JSON 存放 |
| STYLE_CONFIG（styles.py） | 6 種 base_style 各有預設 quality_prefix / negative / LLM 模板 |
| `_inject_loras()`（art_generate.py） | 動態向 ComfyUI workflow 插入 LoraLoader 節點鏈 |
| ComfyUI `/object_info/LoraLoader` | 取得可用 LoRA 清單（autocomplete） |

**ArtStyle 覆寫優先級**
```
手動傳入 art_style_id
  > character.art_style_id（角色預設）
  > project.art_style_id（專案預設）
  > _detect_style()（讀 checkpoint 名稱自動判斷）
```

**操作流程**
1. 點「+ 新增畫風」
2. 填名稱（必填）、說明（選填）
3. 選 Base Style（決定 LLM 模板和預設 quality_prefix）
4. 填 Quality Prefix / Negative（空白 = 繼承 Base Style 預設）
5. 填 Extra Tags（每次生圖固定附加）
6. 新增 LoRA：從 autocomplete 選模型名稱，設定 weight（0–2）
7. 儲存 → 可在角色/專案編輯頁選為預設畫風

**相關 API**
- `GET/POST /api/v1/art-styles`
- `PUT/DELETE /api/v1/art-styles/{id}`
- `GET /api/v1/settings/loras`（LoRA autocomplete 資料來源）

### 功能 6：LoRA 訓練（TrainingTab）

**功能說明**
用自己的插畫作品訓練個人風格 LoRA。上傳圖片、AI 輔助生成 caption、設定超參數、啟動 kohya_ss 訓練、SSE 即時監看進度。訓練完成後自動複製到 ComfyUI loras 目錄，可選擇自動掛載到指定畫風。

**應用技術**
| 技術 | 用途 |
|---|---|
| SQLite（training_images / training_jobs 表） | 圖片與任務持久化 |
| Ollama `qwen2.5vl:7b` | 批次生成圖片 caption（含 trigger word） |
| config_writer.py | 生成 kohya_ss TOML config（自動計算 num_repeats） |
| kohya_ss `train_network.py` | LoRA 訓練引擎（subprocess 呼叫） |
| PyTorch 2.13+cu132 | GPU 訓練（RTX 5070 Ti） |
| SSE（Server-Sent Events） | 即時推送訓練進度到前端 |
| TrainingRunner 抽象層 | 可切換 local（subprocess）/ remote（HTTP agent）模式 |

**漸進訓練策略**
| 量級 | 目的 |
|---|---|
| 5 張 | 驗證 pipeline 能跑通 |
| 10 張 | 確認風格特徵是否被捕捉 |
| 20 張 | 主要品質評估點 |
| 30 張 | 穩定輸出目標 |

**操作流程**
1. 確認頂部狀態列顯示「✓ kohya_ss 就緒」
2. 點「+ 新增圖片」上傳訓練圖（PNG，建議 1024×1024 以上）
3. 點「AI 批次生成 caption」→ Ollama Vision 自動產生描述草稿
4. 逐張確認 / 修改 caption（失焦自動儲存）
5. 填寫訓練設定：任務名稱、Checkpoint、Trigger Word、Rank（預設 32）、學習率、Epochs
6. 選擇綁定畫風（訓練完成後自動掛載 LoRA）
7. 點「建立任務」→「開始訓練」
8. 即時查看進度條（step / total、loss 值）
9. 可展開「查看 Log」檢視 kohya_ss 完整輸出
10. 完成後 .safetensors 自動出現在 ComfyUI loras 目錄

**相關 API**
- `POST /api/v1/training/images/upload`
- `POST /api/v1/training/jobs/{id}/caption-all`
- `POST /api/v1/training/jobs`
- `POST /api/v1/training/jobs/{id}/start`
- `GET /api/v1/training/jobs/{id}/progress`（SSE）
- `GET /api/v1/training/status`（環境檢查）
