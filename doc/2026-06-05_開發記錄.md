## 主題：半身草圖 → 全身生成（擴展功能）

### 需求
上傳半身概念圖（bust/partial）時，生成圖應展示完整全身（頭到腳）。
同時維持 CN 強度 ≥ 0.5，以確保生成圖與草圖外觀 ≥ 70% 相符。

---

## 今日嘗試過程（流水帳）

### 嘗試 1：partial/bust 啟用 CN + `_shrink_for_full_body`（fill=0.60）
- **結果**：生成圖只到大腿，無腳。
- **原因**：CN 0.4 強度 + Canny edges 佔畫布上 60% → SD 認為構圖已完整，下方補背景非腿腳。

### 嘗試 2：調低 fill ratio（0.55）+ 調低 CN weight（0.18）
- **結果**：仍然無腳。
- **原因**：soft CN 引導無法對抗 SD 的構圖慣性；且不符使用者「CN ≥ 0.5」要求。

### 嘗試 3：`end_percent` 早退策略（partial=0.30）
- **機制**：CN 只在降噪前 30% 的步驟作用，後 70% 自由生成腿腳。
- **結果**：生圖出現**左上角 chibi 草稿縮圖 + 3/4 身主圖**。腳仍不完整。
- **發現 #1**：`end_percent=0.30` 理論正確，但構圖仍錯。
- **發現 #2**：小草稿縮圖出現 = `"character design sheet, character reference sheet"` prompt 造成多視圖設計稿格式（SD 的 training 慣例）。

### 嘗試 4（當日最後）：partial/bust 時替換 prompt suffix
- 將 `"character design sheet, character reference sheet"` 改為 `"character illustration, full body portrait"`
- 負向新增 `"multiple views, reference sheet, design sheet, chibi inset, inset image, sketch overlay"`
- **結果**：尚未確認，當日結束。

## 根本矛盾（尚未完全解決）

```
半身草圖的 Canny edges 只存在於畫布上 58%
    ↓
CN 不管多低或多短，都在前幾步把「上半身 = 角色全部」的結構燒進 latent
    ↓
後續步驟無力逆轉構圖 → 腳不出現
```

**唯一可靠解法（尚未實作）：Two-pass generation**

```
Pass 1（8 steps, CN bypass, IPA only）:
    半身草圖 → rough 全身圖（無精度要求，只需有完整的腿腳結構）

Pass 2（20 steps, CN 0.6-0.7）:
    rough 全身圖作為 CN reference（現在 CN ref 是全身的！）
    原始半身草圖作為 IPA reference（外觀一致性）
    → 高精度全身生成圖，CN 高強度有效，腳也在
```

## Two-pass Generation 實作（P0 完成）

### 實作內容

**`backend/app/api/art_generate.py`**

- 新增 `_generate_rough_fullbody()` helper（`generate_character_design` 之前）：
  - 載入工作流、bypass CN、IPA only（上傳 `char_concept_ref.png`）
  - KSampler steps = 8（快速粗稿）
  - 與 Pass 2 相同的 width/height/seed/ipa_weight/lora_list
  - 呼叫 `guardian.request_focus("comfyui")` + `_run_comfyui(wf)` → 返回 bytes

- `generate_character_design`：
  - 補上 `_cn_coverage: str = "full"` 和 `_coverage_end_pct: float | None = None` 預設值
  - `lora_list` 建構移到 `active_wf = state.get_workflow()` 之後（Pass 1 和 Pass 2 共用）
  - coverage 為 partial/bust 時觸發 two-pass：呼叫 `_generate_rough_fullbody`，結果覆蓋 `_cn_ref_bytes`，`_coverage_end_pct` 重設為 None
  - timings 新增 `pass1` key

- `generate_variant_design`：同上

**`frontend/src/components/CharacterTab.jsx`**

- 主角色與變體的耗時區塊均新增 `pass1`（顯示「Pass 1 粗稿」行）
- `keys` 陣列加入 `'pass1'`（納入「其他」計算）

### 觸發條件

- `not is_expression` （全身模式）

全身參考圖（coverage = "full"）不觸發，行為與 6-04 前一致。
