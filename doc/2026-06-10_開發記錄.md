## 主題：UIUX 全面改版（去 AI 味、雙主題、側欄佈局）

## 背景與決策

使用者認為現有 UI「AI 味太重」（深藍紫底 #0f0f17 + 紫 accent #7c6af7 + 霓虹 badge，為生成式 UI 常見配方）。討論後決策：

- **方向**：雙主題（淺色「紙感畫室」為預設 + 深色「中性暖灰」），accent 由紫改朱紅系
- **佈局**：頂部 Tab → 左側導航欄；**歷史記錄保留底部**（使用者指定）；設定維持 modal；各 Tab 內部（左表單/右結果）不動
- 否決：歷史改右側欄

## 今日修改

### 1. `index.css` 雙主題 token 系統
- `:root` = 淺色預設（暖白 #F6F4EE、墨黑字、朱紅 #BC4B24）；`[data-theme="dark"]` = 中性暖灰（#191917、accent #CF5A2E）
- 新增語意 token：`--surface-2`（inset）、`--accent-soft/-2/-border`、`--tint-{blue,green,red,amber,purple,pink}-{bg,fg}`（badge 六色系）、`--code-bg/fg`、`--hover`、`--shadow-modal`
- 字體加 `Noto Sans TC` / `Microsoft JhengHei` fallback

### 2. 寫死色碼清除（211 處 → 0）
- 以 Python 映射批次置換 7 個元件 + App.jsx 的 hardcoded hex/rgba → token
- 保留不動：`#888888`（color picker 預設「值」，非樣式）、`rgba(0,0,0,*)` 與 `rgba(255,255,255,.15)`（壓在圖片上的遮罩，與主題無關）

### 3. `App.jsx` 佈局重構
- 左側欄（196px、sticky）：品牌 + 5 個導航項（active = accent-soft 底）+ 底部（服務狀態燈、主題切換鈕、設定鈕）
- 主欄頂部精簡列：當前功能名稱 + Checkpoint/Workflow 資訊（原 header 內容）
- 歷史記錄維持主欄底部；settings modal / lightbox 不動
- 主題狀態存 `localStorage('craftflow_theme')`，預設 light；`main.jsx` 於 React 掛載前先套 `data-theme` 防閃爍
