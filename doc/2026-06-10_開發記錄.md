## 今日修改

### 1. VRAM Guardian 條件式卸載（P6-1）

**根因**：`request_focus` 原為非黑即白——切到 Ollama 就叫 ComfyUI `/free` 全卸載，切回來 SDXL checkpoint 整顆重載（10~30s）。角色生圖流程「vision → compile（Ollama）→ 生圖（ComfyUI）」每次都付一次重載。

**變更**：
- `core/config.py`：新增 `VRAM_COEXIST_ENABLED`（預設 true，可 env 關閉退回舊行為）、`COMFYUI_REQUIRED_VRAM_GB=8`、`OLLAMA_REQUIRED_VRAM_GB=7`
- `services/ai/vram_manager.py`：新增 `_can_coexist()`——切換前查 ComfyUI `/system_stats`（GPU free / torch reserved）與 Ollama `/api/ps`（size_vram），放得下就只轉移 focus 不卸載；**任何查詢失敗一律回 False 退回舊卸載路徑**（resilient errors）

### 2. vision 抽取結果快取（P6-2）

**根因**：concept images 不變，vision 抽取結果就不變，但每次生成都重打 Ollama vision（5~20s）。

**變更**（`api/art_generate.py`）：
- 新增 `_vision_extract_cached()`：以 image bytes SHA-256 + 模式（coverage/plain N）+ vision model 為 key，in-memory 快取（上限 32 筆，FIFO 淘汰）；錯誤結果（`[...]` 開頭）不快取；cache hit 連 `request_focus("ollama")` 都跳過，ComfyUI 保持 warm
- `generate_character_design` 與 `generate_variant_design` 兩端點改用同一 helper（順帶去掉一段重複碼）

### 3. 修復：檔案掛載同步截斷

編輯過程中 `art_generate.py` 與 `vram_manager.py` 在 sandbox 掛載側發生尾端截斷，已從 git HEAD 補回缺失尾段並全檔重寫修復，`py_compile` 三檔通過，diff 確認僅含本日預期變更。

## 主題五：P4 生圖非同步 job + P5 批次生成

### 10. 非同步 job（P4）

**根因**：`/art/generate` 同步等待回傳 bytes，two-pass / 高解析度 / 批次會撞 HTTP 逾時。

**變更**：
- `services/ai/generation_jobs.py`（新）：in-memory job store（單人本地工具不落 DB），上限 30 筆、完成後保留 30 分鐘；`run_txt2img_job()` 錯誤一律收進 job.error 不外拋；history 記錄用獨立 session
- `api/art_generate.py`：
  - 抽出 `_build_txt2img()`（sync 與 async 共用 workflow 組裝——順帶解決一處重複）
  - 原 `/art/generate` 行為不變（向下相容）

```
POST /art/generate-async                ← 立即回傳 {job_id, seed, batch_size}
GET  /art/jobs/{job_id}                 ← 狀態（queued/running/done/error + elapsed + meta）
GET  /art/jobs/{job_id}/result?index=N  ← 取第 N 張 PNG（X-Seed / X-History-Id headers）
```

### 11. 批次生成（P5 部分）

- `GenerateAsyncRequest.batch_size`（1~8，clamp）→ `EmptyLatentImage.batch_size`
- 同 seed 一次出多張（batch index 變化），一次 checkpoint 載入攤平成本，挑圖效率倍增
- generation_history 記錄含 batch_size

config.py 在掛載側被截斷（VRAM/Personal Style 區塊遺失），已完整重寫並驗證所有設定值。

### 二、草圖問答（無衣服、頭不完整、淡色身體草稿）

| 測項 | CN | 問題 | 結果 |
|---|---|---|---|
| 2.1 | 0.85 | 空白（AI 自動分析） | 合格，但黑白外有彩色躁點 |
| 2.2a | 0.85 | 大小姐服飾/襯衫/長窄裙/馬靴 | **不合格**——CN 太高，服裝指令被草圖壓制 |
| 2.2b | 0.4 | 同上 | 合格以上，但「長窄裙蓋小腿」未實現 |

### 結論與待辦

- **CN 高低兩難**：0.85 鎖構圖→換裝失敗；0.4 換裝成功→裙長控制丟失。「CN≥0.7」限制適用於角色生圖（姿勢/畫風跟隨）；**換裝類用途需例外**，方向：低 CN 換裝 + 提示詞加權（如 `(long narrow skirt covering calves:1.3)`），或 two-pass（低 CN 換裝→高 CN 鎖構圖）。
