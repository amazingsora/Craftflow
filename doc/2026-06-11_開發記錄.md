## 主題二：眼睛優化二次迭代 + 全域預設畫風

### 實測回饋（方案 1+2 失敗）

LoRA 0.6 + extra_tags 後眼睛**未改善且更糟**。修正解讀：BA 風格眼睛（高光）是 LoRA 學來的，
**降權重 = 風格劣化**，方向錯誤。改採：權重回復 0.8，從 FaceDetailer 下手。

### 額外根因發現

`Standard_V35.json` 的 EditDetailerPipe 修復詞為 `{face|face,detailed face}`——
Impact **隨機擇一語法**，一半機率 detailer 只拿到 `face`，解釋了眼睛品質不穩定。

### 1. 眼睛強化版 workflow（原檔不動，可 A/B）

`data/custom_workflows/Standard_V35_EyeFix.json`（複製自 Standard_V35）：
- FaceDetailer denoise 0.26 → **0.35**、steps 14 → **20**
- 修復詞改固定：`[CONCAT] face, detailed face, detailed eyes, beautiful detailed eyes, sharp pupils`
- 風險：denoise 調高可能輕微改變眼型/眼色（與 IPA 外觀跟隨有張力），不行再降回 0.30

### 2. 全域預設畫風（使用者需求：config 一處控制，改一處全部套用）

語意（使用者選定）：**僅作預設值**——優先序
`明確參數 > 角色 > 專案 > 全域預設 > 自動偵測`，沿用既有全域執行期設定模式（state + settings API）。

**後端**：
- `core/state.py`：`_active_art_style_id` + get/set
- `api/settings.py`：`GET/POST /settings/art-style`（POST 驗證 id 存在；null 清除）
- `api/art_generate.py`：新增 `_effective_style_id()` helper，8 個套用點全覆蓋
  （compile-prompt、_build_txt2img（sync+async 共用）、_style_prompts（inpaint/upscale）、
  char-gen 鏈、variant 鏈、img-guide、ipadapter）

**前端**：
- `SettingsTab.jsx`：新增「全域預設畫風」下拉（自包含模式，同全局 LoRA 區塊；localStorage `craftflow_art_style_id`）
- `App.jsx`：開機時以 localStorage 記憶值重新同步後端（後端為執行期狀態，重啟即清空）

### 3. LoRA 回復（已提供指令，使用者本機執行）

`PUT /art-styles/1`：兩顆 LoRA 回 0.8、extra_tags 清空（眼睛 tag 已移到 EyeFix workflow 的 detailer 修復詞，A/B 才乾淨）。

### 採用方案：既有 `.env` Personal Style Preset（零程式碼）

`config.py` 本就有此機制（先前為 BA 用戶設計）：

```ini
PERSONAL_STYLE_ENABLED=true
PERSONAL_STYLE_EXTRA_TAGS=<畫風 tags，換畫風改這行>
PERSONAL_NEGATIVE_ENABLED=true
PERSONAL_NEGATIVE=<個人負向>
```

- 生效範圍：角色生圖 + 變體生圖（`generate_character_design` / `generate_variant_design`）
- 條件：角色綁的畫風 extra_tags 為空才套用（蔚藍檔案 extra_tags 需保持空白）
- LoRA 不在此機制內——換畫風時畫風管理的 LoRA 需一併換
