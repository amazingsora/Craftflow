## 6. Bug 修正：ControlNet 無效果（根本原因追查）

### 症狀
IPA + ControlNet 完全沒有執行，生圖結果與未啟用時相同。

### 追查過程
1. ComfyUI log 無 IPA/ControlNet 模型載入記錄 → 節點根本沒跑
2. 加 DEBUG print 確認：`candidate` 只有 37 個節點（原版），而非 46 個（含 IPA/CN）
3. 進一步確認 `_load_workflow` 實際讀取路徑：

```
載入的是：F:\wk\ComfyUI_portable\ComfyUI\user\default\workflows\craftflow\Standard_V35.json（37節點，原版）
我們改的：F:\wk\Craftflow\data\custom_workflows\Standard_V35.json（46節點，含 IPA+CN）
```

### 根本原因
`.env` 中 `CUSTOM_WORKFLOWS_DIR` 指向 ComfyUI 的 user workflow 目錄，不是 `data/custom_workflows`。  
Craftflow 一直在讀 ComfyUI 原版工作流，從未載入我們的修改版。

### 修法
將修改好的 `Standard_V35.json`（46 節點）複製到 ComfyUI user 目錄：
```
F:\wk\ComfyUI_portable\ComfyUI\user\default\workflows\craftflow\Standard_V35.json
```

**重要提醒：** 未來修改工作流 JSON 必須修改 ComfyUI user 目錄的檔案，或修改後複製過去。
