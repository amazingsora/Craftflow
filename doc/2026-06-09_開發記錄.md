### 2. 建立小說 Tab 骨架（作品 > 集數 > 章節）

**架構**：後端骨架其實已完整存在（`projects`/`chapters` API），今天主要補 `volumes` 中間層並建立前端 UI。

#### 後端新增

| 檔案 | 說明 |
|------|------|
| `models/volume.py` | Volume 模型（project_id, order_index, subtitle） |
| `schemas/volume.py` | VolumeCreate/Response + ReorderItem |
| `api/volumes.py` | Volume CRUD + batch reorder + 集數下建章節 |
| `models/chapter.py` | 加 `volume_id` FK（nullable，不破壞舊資料） |
| `schemas/chapter.py` | ChapterBrief 加 `volume_id` |
| `models/project.py` | 加 `volumes` relationship |
| `core/database.py` | migration 補 `chapters.volume_id` 欄位 |
| `models/__init__.py` | import Volume |
| `main.py` | include volumes router |

**新 API 路由**：
```
GET/POST   /api/v1/projects/{id}/volumes
PUT/DELETE /api/v1/volumes/{id}
PATCH      /api/v1/projects/{id}/volumes/reorder   ← batch reorder
POST       /api/v1/volumes/{id}/chapters
PATCH      /api/v1/volumes/{id}/chapters/reorder   ← batch reorder
```

#### 前端 NovelTab.jsx（全新）

**三欄佈局**：
- Column A（190px）：作品清單 + 建立表單
- Column B（260px）：集數 + 章節 tree，支援 drag & drop 排序
- Column C（flex:1）：章節編輯器

**功能清單**：
- 作品建立（書名 + 作者）
- 集數建立（副標題選填）、刪除（confirm）
- 章節建立（在集數下）、刪除（hover 顯示 ✕）
- 集數拖曳排序（HTML5 drag & drop → `PATCH /volumes/reorder`）
- 章節拖曳排序（同集數內）→ `PATCH /volumes/{id}/chapters/reorder`
- 編輯器：未儲存藍點提示、Ctrl+S 快捷鍵、字數統計
- AI 分析（節奏 + 角色一致性）、改寫建議（左右對比）

---

### 3. 閱讀格式預覽 + 頁碼翻頁

**動機**：使用者希望有小說閱讀版面，以及「下一頁自動儲存翻頁」的寫作流程。

**設計決策**：
- 每個章節 = 一頁（非文字分頁）
- 翻頁邏輯：`turnPage(dir)` → 若有未儲存自動 save → 切換 activeChapter
- 預覽模式：`maxWidth: 620, fontSize: 17, lineHeight: 2.2, textIndent: 2em`，逐行轉為 `<p>` 段落

**UI 變更**（editor header + 新增兩列）：

```
[● dirty dot] [章節標題 input]         [✎ 編輯] [📖 預覽]  ← header
[textarea / 閱讀格式預覽]               ← 主內容（flex:1）
[AI 分析結果 panel]                     ← 條件顯示
[← 上一頁]  [第X集·標題 (N/M)]  [下一頁 →]  ← NEW 翻頁列
[字數]  [分析] [改寫建議] [儲存]        ← footer
```

## 技術備忘

- Volume model 的 `chapters` 關係有 `cascade="all, delete-orphan"`，刪集數會連章節一起刪
- `allChapters` 是每次 render 重新計算的 derived value（非 state），包含 `_vol`、`_volIdx` 輔助欄位
- 章節跨集數拖曳目前不支援（設計決策：保持簡單）
