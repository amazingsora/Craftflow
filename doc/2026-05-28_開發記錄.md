## 1. Bug 修正：概念圖參考功能導致工作流被替換

**問題回報：** 在角色管理中選用 Standard_V35 工作流，開啟「概念圖參考」後，工作流會變成最後使用的模型（ipadapter_txt2img.json）。

### 根本原因

**原因 A — HTTPException 被靜默吞掉：**
`generate_character_design` 和 `generate_variant_design` 中，判斷 active workflow 是否含 IPA 節點的程式段用了 `except Exception`，
而 `_load_workflow` 遇到 UI 格式檔案會 raise `HTTPException(422)` — HTTPException 是 Exception 的子類，所以被靜默捕獲，
直接 fallback 到 `ipadapter_txt2img.json`（系統工作流），使用 global checkpoint（即「最後使用的模型」）。

**原因 B — 工作流無 IPA 節點時強制換工作流：**
即使工作流能正常載入但沒有 IPA 節點，舊邏輯也會強制換成 `ipadapter_txt2img.json`，
用戶選的工作流被棄置。

### 修法（`backend/app/api/art_generate.py`）

```python
# 修前：except Exception 靜默 fallback
try:
    candidate = _load_workflow(active_wf)
    if _wf_has_ipa(candidate): wf = candidate
    else: wf = _load_workflow("ipadapter_txt2img.json")
except Exception:
    wf = _load_workflow("ipadapter_txt2img.json")

# 修後：讓 HTTPException 正常傳出；無 IPA 節點時留在原工作流
candidate = _load_workflow(active_wf)  # HTTPException 不再被吃掉
if _wf_has_ipa(candidate):
    wf = candidate; ipa_used = True   # 有 IPA → 正常注入
else:
    wf = candidate; ipa_used = False  # 無 IPA → 用原工作流，跳過 IPA 注入
                                      # 概念圖視覺分析仍正常運作
```

### Standard_V35.json 是 UI 格式問題

`data/custom_workflows/Standard_V35.json` 是從 ComfyUI 用「Save」匯出的 UI 格式（含 `"nodes"` 陣列），
`_load_workflow` 會 raise `HTTPException(422)`，無法直接使用。

**用戶需要做的操作：**
1. 在 ComfyUI 開啟 Standard_V35 工作流
2. Settings → Enable Dev mode options → 開啟 Dev mode
3. 使用「**Save (API format)**」重新匯出
4. 覆蓋 `data/custom_workflows/Standard_V35.json`

修完後的行為：
- Standard_V35（API 格式，無 IPA 節點）+ 概念圖參考 → 使用 Standard_V35，跳過 IPA 注入，概念圖視覺分析仍有效
- Standard_V35（API 格式，有 IPA 節點）+ 概念圖參考 → 使用 Standard_V35 + IP-Adapter
- UI 格式工作流 → 回傳 422 錯誤給用戶（提示重新匯出），不再靜默 fallback

## 2. Standard_V35.json 加入 IP-Adapter 節點

**目標：** 讓 Standard_V35 工作流支援概念圖參考（IP-Adapter），使生圖盡量貼近上傳的參考圖。

### 修改內容（直接改 JSON）

在 UI 格式的 Standard_V35.json 加入 4 個新節點：

| 節點 ID | 類型 | 說明 |
|---------|------|------|
| 200 | LoadImage | IPA 參考圖（概念圖） |
| 201 | IPAdapterModelLoader | ip-adapter-plus_sdxl_vit-h.safetensors |
| 202 | CLIPVisionLoader | CLIP-ViT-H-14-laion2B-s32B-b79K.safetensors |
| 203 | IPAdapterAdvanced | weight=0.6, weight_type=linear |

**模型鏈修改：**
- 原：Mahiro(68) → KSampler(46)（link 71）
- 改：Mahiro(68) → IPAdapterAdvanced(203) → KSampler(46)（link 71 + 新增 211）

### 額外修正：Node 2 刪除

原工作流的節點 2（Fast Groups Bypasser, rgthree）沒有任何連接，但因 rgthree 未安裝會導致 ComfyUI 報錯「no class_type」。
直接從 nodes 陣列刪除（不影響生圖流程）。

**注意：Standard_V35.json 仍是 UI 格式，需在 ComfyUI 用「Save (API format)」重新匯出後才能被 Craftflow 使用。**

## 3. Bug 修正：無參考圖時 IPA 節點未繞過

**問題：** Standard_V35 加入 IPA 節點後，若用戶沒有開啟「概念圖參考」，工作流仍含活躍的 IPAdapterAdvanced + LoadImage 節點，ComfyUI 因缺少圖片輸入而報錯。

### 修法（`backend/app/api/art_generate.py`）

新增 `_bypass_ipa_nodes(wf)` 函式：
- 找到 IPAdapterAdvanced 節點及其上游模型輸入
- 把所有使用 IPA 輸出的下游節點（如 KSampler）改為直接連接上游模型
- 刪除 IPA 相關節點（IPAdapterAdvanced、LoadImage、IPAdapterModelLoader、CLIPVisionLoader）

在 `generate_character_design` 和 `generate_variant_design` 的 `else` 分支（無參考圖時）加入：
```python
if _wf_has_ipa(wf):
    _bypass_ipa_nodes(wf)
```

最終行為矩陣：

| 情況 | 行為 |
|------|------|
| 有參考圖 + 工作流有 IPA | 正常注入 IP-Adapter |
| 有參考圖 + 工作流無 IPA | 用原工作流，視覺分析仍有效 |
| 無參考圖 + 工作流有 IPA | 自動繞過 IPA 節點，重新接線模型鏈 |
| 無參考圖 + 工作流無 IPA | 正常使用原工作流 |
