## 1. 後端 ComfyUI 生成改用 threadpool

### 問題

`art_generate.py` 多個 `async def` endpoint 內直接呼叫同步 `_run(wf)`，而 `_run()` 會執行 ComfyUI submit、polling、download。這些 `requests` 與 polling 會阻塞 FastAPI event loop，長時間生圖時可能拖慢其他 API。

### 實作

- 在 `backend/app/api/art_generate.py` 引入 `starlette.concurrency.run_in_threadpool`。
- 新增 `_run_comfyui(workflow)` async wrapper。
- 將主要 ComfyUI 生圖路徑改為 `await _run_comfyui(wf)`：
  - `/art/lineart`
  - `/art/generate`
  - `/art/compose`
  - 角色主設計生成
  - 角色變體設計生成
  - `/art/img-guide`
  - `/art/ipadapter`
- WD14 tagger 的 `wait_for_text_result` 也改用 threadpool，避免同步 polling 卡住 event loop。

### 影響

- `_run()` 仍維持同步函式，避免大幅改動 ComfyUI client。
- DB session 沒有跨 thread 傳入 `_run()`，風險較低。
- 生圖時 `/status` 或其他 API 較不容易被長輪詢拖住。

## 2. Ollama client 重複請求邏輯去重

### 問題

`backend/app/services/ai/ollama_client.py` 中 `generate`、`analyze_image`、`analyze_image_bytes`、`analyze_multi_images_bytes` 都重複撰寫：

- `requests.post(f"{OLLAMA_BASE}/api/generate", ...)`
- `raise_for_status()`
- 取出 `response`
- `ConnectionError` 與一般錯誤訊息格式

### 實作

- 新增 `_post_generate(payload, timeout, label)`。
- 文字生成統一使用 label `Ollama`。
- 視覺分析統一使用 label `Vision`。
- 保留原有「錯誤轉成字串回傳」的行為，不改變上層呼叫契約。

### 影響

- 減少重複 try/except。
- 後續若要改成 async HTTP client 或統一 logging，只需改一處。

## 3. ComfyUI polling 抽共用函式並加入 backoff

### 問題

`wait_for_result` 與 `wait_for_text_result` 都在查 `/history/{prompt_id}`，差異只在 outputs 解析方式。原本固定 `time.sleep(2)`，短任務反應較慢，長任務也缺少彈性。

### 實作

- 新增 `_poll_history(prompt_id, timeout, extract)`。
- 新增 `_extract_images(outputs)` 與 `_extract_texts(outputs)`。
- polling interval 從 `0.5s` 起跳，逐步乘以 `1.5`，上限 `3s`。

### 影響

- 圖片與文字結果 polling 共用同一套逾時與輪詢流程。
- 短任務可更快返回，長任務不會過度密集打 ComfyUI。

## 4. 前端新增共用 API 工具層

### 問題

前端多個 component 直接使用 raw `fetch('/api/v1/...')`，錯誤處理與 base URL 分散。`CharacterTab.jsx` 雖已有本地 `apiFetch` / `apiDelete`，但仍含本地 `API = '/api/v1'` 與少數 raw fetch。

### 實作

新增：

- `frontend/src/api/client.js`
  - `request(path, opts)`
  - `apiGet`
  - `apiPostJson`
  - `apiPostForm`
  - `apiPostBlob`
  - `apiDelete`
  - `apiUrl`
- `frontend/src/api/endpoints.js`
  - 集中繪圖相關 endpoint 常數。
- `frontend/src/api/useAsync.js`
  - 共用 loading / error / run wrapper。

### CharacterTab 調整

- 移除本地 `const API = '/api/v1'`。
- 本地 `apiFetch` 改包共用 `request()`。
- 本地 `apiDelete` 改用共用 `apiDelete()`。
- 圖片 URL 與下載 URL 改用 `apiUrl()`。
- 角色生成與變體生成因需讀取 headers/blob，改使用 `request()` 保留 `Response`。

### GenerateTab 調整

- `compile-prompt` 改用 `apiPostJson(EP.compilePrompt, ...)`。
- `ipadapter` / `img-guide` 改用 `apiPostForm(...)`。
- `generate` 保留 `Response` blob 需求，改用 `request(EP.generate, ...)`。
- `loading` / `optimizing` / `error` 改由兩個 `useAsync()` task 管理。
