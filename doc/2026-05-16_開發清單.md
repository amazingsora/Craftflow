# Craftflow 開發清單 (2026-05-16)

這是針對分頁衝突、GPU 效能優化、角色管理功能的開發記錄。

---

## 🔍 問題分析 (Issues)
- [ ] **分頁衝突 (Tab Contention)**：
    - 現象：A 分頁產生說明，B 分頁內容錯亂。
    - 原因：`localStorage` 全域 key (`gen_promptZh` 等) 被多分頁共享。
- [x] **GPU 資源競爭**：
    - 現象：同時操作時 AI 回傳內容不完整或錯誤。
    - 原因：5070 Ti 顯存同時處理多個 Vision/Diffusion 請求。
    - **已解決**：實作 VRAM Guardian + Ollama keep_alive 卸載機制。

---

## ✅ 已完成 (2026-05-16)

### GPU / VRAM 優化
- [x] **VRAM Guardian 完整實作**
    - `vram_manager.py`：`_unload_ollama()` 呼叫 `/api/ps` 列出已載入模型，再以 `keep_alive: 0` 逐一卸載。
    - `_unload_comfyui()` 呼叫 ComfyUI `/free` 端點清除 diffusion 模型。
- [x] **Guardian 接入所有 AI 路由（Option A：Router 層）**
    - `ai_text.py`：4 個端點轉 `async def`，呼叫前加 `await guardian.request_focus("ollama")`。
    - `ai_art.py`：5 個端點轉 `async def`，同上。
    - `art_generate.py`：compile_prompt → ollama focus；lineart/generate → comfyui focus；compose → 先 ollama 再 comfyui。
- [x] **清理 DEBUG print**：`art_service.py`、`art_generate.py` 所有 `print(f"DEBUG: ...")` 改為 `logger.debug/info`，移除計時變數。

### 角色管理功能擴充
- [x] **作品重新命名**：`CharacterListView` 新增「編輯作品」按鈕，inline 表單可修改名稱、作者、類型、狀態、簡介；`ProjectsView` 每張卡片加「編輯」按鈕。
- [x] **角色重新命名**：`CharacterDetailView` 角色名稱欄位改為可編輯，與其他欄位一起透過「儲存所有變更」寫回。
- [x] **角色代表色**：新增 `<input type="color">` 色盤 + 十六進位色號顯示，建立與編輯頁均支援，清除鍵可移除顏色。

### 勢力系統（Faction）— 全新功能
- [x] **資料模型**：新增 `Faction` ORM 模型（`factions` 表），與 `Character` 多對多關聯（`character_factions` join table），`Character.faction_ids` 屬性供 Pydantic 序列化。
- [x] **後端 API** (`/backend/app/api/factions.py`)：
    - `GET /projects/{id}/factions`：列出勢力（含成員）
    - `POST /projects/{id}/factions`：建立勢力
    - `PUT /factions/{id}`：重新命名
    - `DELETE /factions/{id}`：刪除（含 cascade 成員關聯）
    - `POST/GET /factions/{id}/thumbnail`：上傳 / 取得縮圖
    - `POST /factions/{id}/members/{char_id}`：加入成員
    - `DELETE /factions/{id}/members/{char_id}`：移出成員
- [x] **前端 UI**：
    - `CharacterListView`：勢力磚（虛線邊框）與無勢力角色同層顯示；「+ 新增勢力」inline 建立；勢力磚顯示縮圖、名稱、成員數、刪除按鈕。
    - `FactionView`：勢力詳情頁，支援重新命名、縮圖上傳（拖曳/點擊）、加入/移出成員。
    - `CharacterDetailView`：「所屬勢力」區塊，顯示勢力 chip（× 可離開）、「+ 加入勢力」下拉選單。

### 刪除確認保護
- [x] **`DeleteConfirm` 元件**：所有破壞性刪除操作需輸入對應名稱才能啟用確認按鈕，適用於角色刪除、勢力刪除。

### Bug 修復
- [x] **作品編輯靜默失敗**：補上 `saveError` 狀態顯示，修正 catch 區塊，名稱空值前置驗證。

### 提示詞引擎優化 (Prompt Engine Optimization)
- [x] **消除標籤冗餘**：修正 Few-shot 範例，移除重複標籤。
- [x] **多樣化範例導引**：為所有模型模板增加多樣化範例，避免 AI 對單一範例產生過度擬合。
- [x] **異色瞳精確度提升**：加入帶方向性的標籤範例（left/right），強化細節描述。
- [x] **抑制幻覺細節 (Anti-Hallucination)**：加入 `STRICT` 規則，防止 AI 擅自添加未提及的服裝或背景。
- [x] **性別與格式修正**：強制 `1boy/1girl` 開頭並禁止非標籤格式（如資料卡格式），解決男性角色偏見問題。
