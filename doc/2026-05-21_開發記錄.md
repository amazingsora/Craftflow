## 1. 設定功能（Settings Modal）

### 問題
ComfyUI 離線時 `/api/v1/settings/checkpoints` 回 503，前端清單為空，Header 的 Checkpoint 選單整個隱藏。使用者每次重連都要重新設定。

### 解決方案
將設定從 Tab 改為**右上角齒輪按鈕 → Modal 彈窗**，提供：

**生成模式切換（Radio）**
- `Checkpoint + LoRA 模式`：使用系統 text_to_image.json，Checkpoint 可選，LoRA 注入有效
- `自訂 Workflow 模式`：使用使用者匯出的 API 格式 JSON，Checkpoint/LoRA 由 workflow 本身決定

**Checkpoint 設定區塊**
- ComfyUI 在線 → 下拉選擇清單
- ComfyUI 離線 → 手動輸入名稱 + 儲存按鈕（記憶到 localStorage，連線後自動生效）

**全局 LoRA 設定區塊**（Checkpoint 模式才顯示）
- ComfyUI 在線 → 下拉選擇 LoRA 清單 + 強度滑桿
- ComfyUI 離線 → 手動輸入 LoRA 名稱
- 套用後每次生成都會注入此 LoRA（疊加在 art_style LoRA 之前）

**Workflow 設定區塊**（Workflow 模式才顯示）
- 有自訂 workflow → 下拉選擇 + 清單顯示（標示使用中）
- 無自訂 workflow → 說明如何從 ComfyUI 匯出 API 格式 JSON

**Header 常駐顯示**
- 不管 ComfyUI 是否在線，Header 都顯示目前模式名稱與 active 值
- 點擊齒輪 ⚙ 開啟設定 Modal

### 技術決策
- `generationMode` 只在前端管理，後端不感知——切換模式時同步 workflow 值到後端
- Workflow 模式 → 切換時 POST `/settings/workflow` 設為 active workflow
- Checkpoint 模式 → 切換時 POST `/settings/workflow` 重設回 `text_to_image.json`

### 修改檔案
- 新增 `frontend/src/components/SettingsTab.jsx`
- `frontend/src/App.jsx`：齒輪按鈕、Modal、generationMode state、localStorage 持久化
- `backend/app/core/state.py`：加入 `_active_lora {name, strength}` 狀態
- `backend/app/api/settings.py`：加入 `GET/POST /settings/lora` 端點，`GET /loras` 回傳 active
- `backend/app/api/art_generate.py`：generate 時先注入 global LoRA，再疊加 art_style LoRA

## 3. Workflow 模式 Prompt 注入問題

### 問題
使用者的 `Anima_workflow.json` 採用 Anima/FLUX 分離架構（UNETLoader + CLIPLoader + VAELoader），CLIPTextEncode 節點已有真實文字，沒有 `__POSITIVE__`/`__NEGATIVE__` 佔位符，導致 GenerateTab 輸入的 prompt 完全沒有注入。

### 解決方案
新增 `_inject_prompts()` helper，三段優先級判別：
1. `__POSITIVE__` / `__NEGATIVE__` 佔位符（系統內建 workflow）
2. `_meta.title` 包含 "positive" / "negative" 關鍵字（使用者自訂 workflow，如 Anima）
3. 依 node ID 排序後，第一個 CLIPTextEncode = positive，第二個 = negative（兜底）

Anima workflow 的 node 5 標題為 `"Positive Prompt"`、node 6 為 `"Negative Prompt"`，命中第 2 優先級。

## 4. Workflow 切換未同步問題

### 問題
前端載入時，`target` 計算 fallback 為 `data.active`（後端預設 `text_to_image.json`），但此值不在自訂清單（只有 `Anima_workflow.json`）。`<select value>` 與選項不一致時，瀏覽器視覺顯示第一個 option，但不觸發 `onChange`，React state 停留在舊值。

### 解決方案
- App.jsx `useEffect`：target fallback 改為 `list[0]`（確保在清單內）
- SettingsTab `useEffect`：`activeWorkflow` 不在 `workflows` 清單時，自動呼叫 `onWorkflowChange(workflows[0])`

## 5. Workflow 選擇器優化：分離 custom/system 目錄

### 問題
`CUSTOM_WORKFLOWS_DIR` 與 `_SYSTEM_WORKFLOW_DIR` 指向同一目錄，下拉選單混入 `text_to_image.json`、`sketch_to_lineart.json` 等系統檔案。

### 解決方案
- `config.py`：`CUSTOM_WORKFLOWS_DIR` 預設改為 `data/custom_workflows/`（專案根目錄下）
- `settings.py`：`list_workflows()` 加入 `mkdir(parents=True, exist_ok=True)`，首次存取自動建立
- `SettingsTab.jsx`：指引文字改為 `data/custom_workflows/`
- 使用者自訂 workflow 放 `data/custom_workflows/`，系統 workflow 留 `tools/Craftflow/diffusion/workflows/`

## 8. Workflow 初始化同步 Bug 修復

### 問題
App.jsx 初始化時：`GET /settings/workflows` 回傳 `["Anima_workflow.json"]`，後端 active = `text_to_image.json`（不在清單內），target fallback 為 `list[0] = Anima_workflow.json`，POST 更新後端。導致即使 UI 在 **Checkpoint 模式**，後端 `state.get_workflow()` 已被設成 Anima，generate 走到錯誤 workflow。

### 修復
`App.jsx` useEffect 中，讀 `localStorage.getItem('craftflow_gen_mode')`，只有 `workflow` 模式才把自訂 workflow 推給後端；`checkpoint` 模式強制保持後端為 `text_to_image.json`。
