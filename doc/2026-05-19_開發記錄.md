## 2. Phase 2 — Prompt Engine 整合

### 覆寫層設計

```
effective base_style     = art_style.base_style     (若有)   else _detect_style() (讀 checkpoint)
effective quality_prefix = art_style.quality_prefix (若非空) else STYLE_CONFIG[base_style]["quality_prefix"]
effective negative       = art_style.negative       (若非空) else STYLE_CONFIG[base_style]["negative"]
extra_tags               = 附加在 final_positive 尾端（每次生成都加）
```

空字串 = 不覆寫，繼承基底預設。

### 異動

**`backend/app/services/ai/prompt_engine/compiler.py`**
- `compile()` 新增 `quality_prefix_override: str | None` 和 `negative_override: str | None` 兩個可選參數

**`backend/app/api/art_generate.py`**
- 新增三個 helper：
  - `_resolve_style(art_style, workflow)` — base_style 覆寫 > checkpoint 偵測
  - `_compile_overrides(art_style)` — 回傳傳入 `compile()` 的 kwargs dict
  - `_extra_tags(art_style)` — 取 extra_tags 字串

受影響端點（加入 `art_style_id` 支援）：

| 端點 | 接收方式 |

## 3. Phase 3 — LoRA 注入（ComfyUI Workflow）

### `_inject_loras(wf, loras)` 函數

加入 `art_generate.py`，邏輯：

1. 過濾空 model 名稱（靜默跳過）
2. 找到 `CheckpointLoaderSimple` 節點 ID 作為鏈起點
3. 逐一插入 `LoraLoader` 節點（`_lora_0`, `_lora_1`, ...）形成鏈式結構
4. 重新連接所有 `CLIPTextEncode.clip` 和 `KSampler.model` 到最後一個 LoRA 節點輸出
5. 若 workflow 無 `CheckpointLoaderSimple`（如 lineart），log warning 並跳過

呼叫位置：`/art/generate`、`generate_character_design`、`generate_variant_design` 三個端點的 `_load_workflow()` 之後。

### GET /api/v1/settings/loras

加入 `settings.py`，從 ComfyUI `/object_info/LoraLoader` 解析可用 LoRA 清單，與 checkpoints 端點相同結構。

## 資料結構參考

### `art_styles` 資料表

```sql
CREATE TABLE art_styles (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            VARCHAR(100) NOT NULL UNIQUE,
    description     TEXT,
    base_style      VARCHAR(30) DEFAULT 'sdxl',
    quality_prefix  TEXT DEFAULT '',
    negative        TEXT DEFAULT '',
    extra_tags      TEXT DEFAULT '',
    loras           TEXT DEFAULT '[]',   -- JSON array [{model, weight}]
    created_at      DATETIME,
    updated_at      DATETIME
);
```

### LoRA 節點鏈（注入後的 workflow 結構）

```
CheckpointLoaderSimple("1")
    ↓ model[0], clip[1]
LoraLoader("_lora_0") lora_name=A, strength=0.8
    ↓ model[0], clip[1]
LoraLoader("_lora_1") lora_name=B, strength=0.6
    ↓ model[0]          ↓ clip[1]
KSampler              CLIPTextEncode ×2
```

## 5. Phase 5 — 畫風套用到角色 / 專案

### 目標

角色可指定預設畫風，生成時自動套用，不需每次手動傳 `art_style_id`。

### 優先級規則

```
手動傳入的 art_style_id（query param）
  > character.art_style_id（角色預設畫風）
  > project.art_style_id（專案預設畫風）
  > _detect_style()（全域 checkpoint 偵測，原有邏輯）
```

### 異動檔案

**`backend/app/models/character.py`**
- 新增 `art_style_id: Mapped[Optional[int]]`，FK → `art_styles.id`，nullable

**`backend/app/models/project.py`**
- 新增 `art_style_id: Mapped[Optional[int]]`，FK → `art_styles.id`，nullable
- 補上 `Integer`, `ForeignKey` import

**`backend/app/schemas/character.py`**
- `CharacterCreate` / `CharacterUpdate` / `CharacterResponse` 均加入 `art_style_id: Optional[int] = None`

**`backend/app/core/database.py`**
- `_migrate()` 新增：
  - `_add_columns("projects", [("art_style_id", "INTEGER")])`
  - `_add_columns("characters", [("art_style_id", "INTEGER")])`
- 後端重啟時自動 ALTER TABLE，現有資料庫無需手動操作

**`backend/app/api/art_generate.py`**
- `generate_character_design` 端點：query param `art_style_id` 為 None 時，依序讀取 `character.art_style_id` → `project.art_style_id`，再 fallback 到 `_detect_style()`
- `generate_variant_design` 端點：同上邏輯

```python
# 兩個端點都加入此 priority chain（位於 character 載入之後）：
if art_style_id is None and character.art_style_id:
    art_style_id = character.art_style_id
if art_style_id is None:
    from app.models.project import Project as _Project
    proj = db.get(_Project, character.project_id)
    if proj and proj.art_style_id:
        art_style_id = proj.art_style_id
```

**`frontend/src/components/CharacterTab.jsx`**（`CharacterDetailView` 元件）
- 新增 state：`artStyleId`（初始值 `initChar.art_style_id`）、`artStyles`（畫風清單）
- 新增 `useEffect`：頁面載入時呼叫 `GET /api/v1/art-styles` 取得所有畫風
- `saveFields` 的 PUT body 加入 `art_style_id: artStyleId || null`
- 表單新增「預設畫風」`<select>`（性別 picker 下方、所屬勢力上方）：
  - 預設選項為「（不設定，依全域 checkpoint 自動偵測）」
  - 選項格式：`{name} [{base_style}]`

### 使用方式

1. 後端重啟（`_migrate()` 自動加欄位）
2. 進角色編輯頁 → 性別選擇器下方出現「預設畫風」下拉
3. 選取畫風後按「儲存所有變更」
4. 之後點「生成人設圖」不需另外傳 `art_style_id`，後端自動套用

## 6. Phase 7 技術調查（討論紀錄）

### 背景

使用者誤以為 LoRA 訓練只需要 JPG 圖片。本次討論釐清了訓練資料的完整結構與架構障礙。

### 關鍵發現：圖文配對（Caption 系統）

LoRA 訓練不只是丟圖片，每張圖需要配對同名 `.txt` 描述檔：

```
dataset/img/10_myartstyle/
  img001.png
  img001.txt    ← "1girl, silver hair, red eyes, school uniform"
```

- Caption 描述**畫面內容**，絕對不描述風格本身
- 每個 caption 最前面加 trigger word（例如 `myartstyle`）
- `10_` 前綴是「每 epoch 重複次數」，圖少時調高補足訓練量

### 關鍵發現：Docker 障礙

後端若跑在 Docker 內，**無法直接 subprocess 呼叫 Windows 宿主機的 kohya_ss**。

評估後建議：**後端改跑 Windows 本機**（`uvicorn` 直接在 PowerShell），省去 Docker 層，對個人工具最簡單。
