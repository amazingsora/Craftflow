## 2. Phase 2 — Prompt Engine 整合

### 覆寫層設計

```
effective base_style     = art_style.base_style     (若有)   else _detect_style() (讀 checkpoint)
effective quality_prefix = art_style.quality_prefix (若非空) else STYLE_CONFIG[base_style]["quality_prefix"]
effective negative       = art_style.negative       (若非空) else STYLE_CONFIG[base_style]["negative"]
extra_tags               = 附加在 final_positive 尾端（每次生成都加）
```

空字串 = 不覆寫，繼承基底預設。

## 3. Phase 3 — LoRA 注入（ComfyUI Workflow）

### `_inject_loras(wf, loras)` 函數

加入 `art_generate.py`，邏輯：

1. 過濾空 model 名稱（靜默跳過）
2. 找到 `CheckpointLoaderSimple` 節點 ID 作為鏈起點
3. 逐一插入 `LoraLoader` 節點（`_lora_0`, `_lora_1`, ...）形成鏈式結構
4. 重新連接所有 `CLIPTextEncode.clip` 和 `KSampler.model` 到最後一個 LoRA 節點輸出
5. 若 workflow 無 `CheckpointLoaderSimple`（如 lineart），log warning 並跳過

呼叫位置：`/art/generate`、`generate_character_design`、`generate_variant_design` 三個端點的 `_load_workflow()` 之後。

### GET /api/v1/settings/loras

加入 `settings.py`，從 ComfyUI `/object_info/LoraLoader` 解析可用 LoRA 清單，與 checkpoints 端點相同結構。

## 6. Phase 7 技術調查（討論紀錄）

### 背景

使用者誤以為 LoRA 訓練只需要 JPG 圖片。本次討論釐清了訓練資料的完整結構與架構障礙。

### 關鍵發現：圖文配對（Caption 系統）

LoRA 訓練不只是丟圖片，每張圖需要配對同名 `.txt` 描述檔：

```
dataset/img/10_myartstyle/
  img001.png
  img001.txt    ← "1girl, silver hair, red eyes, school uniform"
```

- Caption 描述**畫面內容**，絕對不描述風格本身
- 每個 caption 最前面加 trigger word（例如 `myartstyle`）
- `10_` 前綴是「每 epoch 重複次數」，圖少時調高補足訓練量

### 關鍵發現：Docker 障礙

後端若跑在 Docker 內，**無法直接 subprocess 呼叫 Windows 宿主機的 kohya_ss**。

評估後建議：**後端改跑 Windows 本機**（`uvicorn` 直接在 PowerShell），省去 Docker 層，對個人工具最簡單。

## 7. 提示詞引擎重構與精確度優化 (Gemini CLI)

### 背景
為了提升提示詞生成的穩定性與可維護性，並解決視覺分析（Vision）在單色背景下產生的「幻覺」與文字描述衝突的問題，由 Gemini CLI 進行了核心提示詞引擎的深度重構。

### 7-1. 第一階段：架構重構與衝突仲裁
- **結構化與型別安全 (`styles.py`)**：引入 `StyleConfig` Pydantic 模型，自動將 `quality_prefix` 標籤注入 `banned_tags`，徹底消除品質標籤重複堆疊。
- **模板模組化 (DRY)**：將顏色翻譯表、格式禁令等提取至共通規則，各模型模板透過字串格式化組合。
- **確立優先權規則**：在 LLM 模板中定義 `CONFLICT` 規則，確保「優先採用（外貌與個性）」欄位的文字描述絕對壓過「視覺特徵」的自動提取。
- **視覺分析「背景防禦」**：更新 `art_generate.py` 的視覺提取提示詞，加入強烈警告要求 AI 忽略單色背景色，避免背景污染提示詞。

### 7-2. 第二階段：標籤排序與權重強化
- **詞庫外部化 (`lexicon.py`)**：將所有顏色映射、正則表達式、以及髮型/眼部特徵集移出邏輯層，提升擴展性。
- **標準化標籤排序 (Standardized Ordering)**：實現決定性的標籤重排邏輯。確保 Prompt 遵循：`品質前綴 > 主體 (1girl/solo) > 硬性錨點 (Anchors) > 其他細節 > Meta 標籤`。
- **權重強化 (SD Weighting)**：編譯器自動為「硬性錨點」加上權重語法（如：`(short hair:1.1)`），增強 Stable Diffusion 對手寫描述的遵循度。
- **增強型正則提取**：優化髮型捕捉正則，完美支援「短褐色頭髮」等複雜語序，並在發現「短髮」錨點時強制從 LLM 輸出中清除所有衝突的長髮標籤。
