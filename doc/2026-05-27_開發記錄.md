## 1. 流程分析：上傳圖片 → 提示詞 → 生圖完整路徑

釐清現有四個階段：

| 階段 | 技術 | 說明 |
|------|------|------|
| A 視覺分析 | `qwen2.5vl:7b` | 概念圖 → 中文特徵短語（髮色/眼色/服裝） |
| B 中文描述組合 | Python | `raw_desc` = 角色資料 + 視覺特徵 |
| C 使用者輸入 | GenerateTab | 手打或從角色卡/ComposeTab 帶入 |
| D 語言模型編譯 | `dolphin-llama3` + `compiler.py` | 中文 → Danbooru tags，含防幻覺過濾 |

**識別痛點：**
1. 視覺模型輸出純黑盒，用戶看不到中文描述
2. 語言模型輸出的英文 prompt 前端幾乎不展示
3. 手打提示詞無 Danbooru 自動補全輔助
4. 負向提示詞完全靠 style 預設，無個人化空間

## 2. Bug 修正：text model 初始化 fallback 錯誤

**問題：** `App.jsx:388` 初始化 text model 時呼叫 `vision-models` endpoint，
`data.default` 回傳 `DEFAULT_VISION_MODEL`，導致首次使用時 text model fallback 設成視覺模型的預設值。

**修法：** 改用 `Promise.all([vision-models, text-model])` 同時取模型清單與正確 default。

**檔案：** `frontend/src/App.jsx:388`

## 5. Personal Style Config 系統

**需求：** 個人風格（Blue Archive 風格 + 自訂負向）本機預設 true、git 上 false。

**實作方式：** `.env`（已在 .gitignore）+ `config.py` 讀取。

### 新增環境變數

```
PERSONAL_STYLE_ENABLED=true/false
PERSONAL_STYLE_EXTRA_TAGS=...
PERSONAL_NEGATIVE_ENABLED=true/false
PERSONAL_NEGATIVE=...
```

### 優先順序邏輯

```
art_style.extra_tags 有值 → 優先
                    沒值 → PERSONAL_STYLE_ENABLED=true 時，套用個人風格標籤

art_style.negative 有值  → 優先
                   沒值  → PERSONAL_NEGATIVE_ENABLED=true 時，套用個人負向
```

### 個人設定值（本機 .env）

**PERSONAL_STYLE_EXTRA_TAGS：**
```
Blue Archive style, anime game illustration, clean lineart, soft shading,
large expressive eyes, delicate face, pale skin, light blush,
pastel tone, crisp edges, bright highlights, high detail
```

**PERSONAL_NEGATIVE：**
```
lowres, bad quality, worst quality, sketch, rough lineart, photorealistic,
3d, realistic, heavy rendering, oily skin, blurry face, deformed hands,
extra fingers, bad anatomy, watermark, signature, text
```
