## 技術決策

### `_resolve_conditioning` 設計
ControlNetApplyAdvanced 有兩個 conditioning 輸出：slot 0 = positive conditioned、slot 1 = negative conditioned。
Pass 1 從 KSampler 邊的 `[node_id, output_slot]` 出發，根據 output_slot 遞迴穿透中間節點，確保即使有 CN 注入時也能找到正確的 CLIPTextEncode。

### global LoRA 優先順序
與 `/art/generate` 對齊：global_lora（state） → art_style.loras，順序一致。

## 生成耗時修正

### 問題
視覺分析 3.8s + 提示詞編譯 2.8s + ComfyUI 33.8s = 40.4s，但總計顯示 57.9s，落差 17.5s 未計入。

### 原因
後端 `timings` 字典未追蹤以下耗時：
- `_detect_body_coverage`（Ollama 視覺推論，全身模式才跑）→ 實測 ~10s
- ComfyUI 圖片上傳（IPA + CN ref 各一次）→ 實測 ~4s
- `guardian.request_focus` 等待 + 其他準備工作 → ~2s（以「其他」補足）

### 修改內容

**後端 `art_generate.py`（char + variant 各一套）**
- 新增 `body_coverage` timing：`_detect_body_coverage` 前後計時
- 新增 `upload` timing：IPA 上傳計時，CN 上傳累加（`timings.get("upload", 0.0) + ...`）
- 新增 `models` dict：`{ vision, text, workflow }` 於 return 前寫入

**前端 `CharacterTab.jsx`（主角色 + 變體）**
- 每行改為三元組 `[label, model, sec]`，model 顯示為標籤後小字
- 新增「姿態偵測」、「圖片上傳」兩行
- 加「其他」行 = `total - sum(已知項)`，差距 > 0.5s 才顯示
- 總計維持後端 wall-clock `total`（準確值）
