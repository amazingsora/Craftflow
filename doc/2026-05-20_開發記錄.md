## 2. 草圖意見功能優化 + Bug 修復

### 背景
手動調整 3 個檔案（`art_service.py`、`compiler.py`、`comfyui_client.py`）後，草圖意見功能正常，但引入了多個 breaking change。

### 優化內容（保留）

**`art_service.py` — 草稿/完稿/線稿配色 Prompt 升級**
- `_SKETCH_PROMPT`：診斷語言更精準（「輪廓線需要加強」→「輪廓線交叉或斷裂」）；回應結構從「優先修正清單 + 值得保留」改為「具體修改步驟（2-3 點）」，方向從評審轉向指導
- `_FINISHED_PROMPT`：新增骨架結構審查維度（視覺衝擊力、光影色彩、骨架人體、細節精緻度、綜合改進方向）
- `_LINE_COLOR_PROMPT`：從泛用配色建議改為 3 套完整方案（主色 60%/輔色 30%/點綴色 10% + 氛圍描述）
- `analyze_composition_bytes`：格式標記改為 `[ADVICE]`/`[PROMPT]`，加入硬性規則限制 SDXL Prompt 不腦補場景，temperature 0.4→0.2，num_predict 350→450

**`compiler.py` — 防幻覺與情緒召回**
- 新增 `_clean_clothing_hallucinations()`：偵測休閒服關鍵字時自動移除 LLM 腦補的正裝 tag
- 新增情緒召回機制：文字含笑/微笑時，強制插入 `smile` tag
- `_sanitize_to_list` 改為支援 `\n` 和 `#` 分隔
- `_reorder_tags` 新增服裝分類（clothing 排在 anchors 之後）
